# Noiseglass

Noiseglass is an AI-powered support intelligence platform that transforms noisy customer support tickets into ranked, actionable product insights. It uses the Claude API to classify and summarize support issues while relying on deterministic Python code to perform clustering and trend analysis, ensuring all numerical calculations remain accurate and reproducible.

Built as a portfolio project, Noiseglass demonstrates how modern AI can automate support operations by identifying recurring customer problems, prioritizing emerging trends, and surfacing insights that product and engineering teams can immediately act on.

## How it works

1. **Ticket ingestion**: Noiseglass can analyze both synthetic support datasets and real GitHub Issues collected through the GitHub REST API. This allows the application to demonstrate its workflow using realistic sample data while also supporting live repositories.

2. **Pass 1, AI classification**: Claude reads batches of incoming tickets and classifies each one into a normalized category while generating a concise issue summary. Batching minimizes API requests and improves processing efficiency.

3. **Pass 2, deterministic analysis**: Python groups related tickets into clusters and calculates week-over-week trend changes, frequency, and severity metrics. All arithmetic and aggregation are performed in code rather than by the language model to ensure consistent, trustworthy results.

4. **Pass 3, AI summarization**: Claude receives the computed statistics and produces concise signal summaries, recommended actions, and severity assessments based on the verified data generated by the application.

5. **Visualization**: The frontend presents a live dashboard where incoming tickets are transformed into prioritized product signals. Each signal includes trend information, representative tickets, severity, and recommended next steps.

## Technology

* **Frontend:** React and Vite
* **Backend:** FastAPI
* **AI:** Claude API
* **Database:** PostgreSQL with SQLAlchemy for persistent storage of tickets, analyses, and application data
* **Migrations:** Alembic
* **Deployment:** Railway
* **Version Control:** Git and GitHub, using Git Bash and the VS Code integrated terminal throughout development
* **External Integrations:** GitHub REST API for retrieving live GitHub Issues

## Running locally

### Backend

```bash
cd backend
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python -m uvicorn main:app --reload --port 8000
```

To verify the complete pipeline independently:

```bash
python test_pipeline.py
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Open the local URL displayed by Vite. API requests are proxied to the FastAPI backend running on port 8000.

## Features

* AI-powered support ticket classification
* Live GitHub Issue ingestion through the GitHub API
* Support for synthetic datasets for testing and demonstrations
* Trend detection using deterministic Python calculations
* AI-generated product insights and recommended actions
* PostgreSQL persistence with SQLAlchemy ORM
* Expandable dashboard with ticket details, trend visualization, and severity indicators
* Cached analysis results to reduce repeated Claude API costs

## Design philosophy

Support teams receive hundreds of repetitive tickets that often obscure the problems that matter most. Triage is designed to reduce that noise by combining AI classification with deterministic analytics, allowing product and engineering teams to quickly identify emerging customer issues, understand their impact, and prioritize fixes based on real support trends instead of intuition.
